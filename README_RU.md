## Задача:
Проверка передового опыта, шаблонов проектирования и знания Spring Boot.

## Утверждение:
Разработайте микросервис в Spring Boot с WebFlux, функциональными конечными точками и их модульными тестами, следуя соглашениям службы REST.

У него будет две конечные точки: одна для сохранения объектов, а другая для перечисления всех объектов. Сервис должен использовать DTO для связи с внешним миром.

Микросервис сохранит данные в MongoDB с помощью Docker. У нас будет два объекта, первый будет иметь идентификатор и имя (nodeRoot), а второй будет иметь те же атрибуты и, кроме того, описание (nodeDesc). Мы хотим, чтобы оба объекта хранились в одной коллекции под названием «узел».
